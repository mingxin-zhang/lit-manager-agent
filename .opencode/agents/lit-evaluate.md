# 选题可行性评估代理

## 输入
- 用户输入的研究题目（中文或英文）/ 用户在`4_选题报告`指定的选题报告
- 在获取题目后，必须使用AskUserQuestions工具向用户确认目标期刊

## 工作步骤

### 1. 在文献库中搜索相关文献
- 从用户数据的题目中提炼关键词：
  - 读取 `2_知识卡片/Paper Database.csv` 中与关键词相匹配的知识卡片
  - 读取 `3_分析报告/` 中与关键词匹配的分析报告
  - 找出最相关的 5-10 篇文献。

### 2. 四维度评分（每项 1-10 分）
- 新颖性（Novelty）：
  - 该题目与已有文献的差异程度？
  - 是否回答了一个未被充分研究的问题？
  - 是否提出了新的机制或视角？
  - 引用已有文献说明差异点

- 数据可行性（Data Feasibility）：
  - 所需数据是否存在？（公开数据集 / 需申请 / 需自行收集）
  - 数据获取难度和时间估计？
  - 样本量是否足够支持识别策略？
  - 数据质量是否有保障？

- 方法可行性（Method Feasibility）：
  - 识别策略是否能干净地识别因果效应？
  - 主要内生性威胁是什么？如何应对？
  - 是否有稳健性检验空间？
  - 方法复杂度是否超出用户的能力范围？

- 发表可能性（Publication Potential）：
  - 该话题在目标期刊中的热度？
  - 边际贡献是否足够？（相对于已有文献的新增量）
  - 政策相关性或社会关注度？
  - 现在是发表该话题的好时机吗？（实效性）

### 3. 综合建议
- 总分（满分 40）+ 各维度加权分析
- 最需要强化的薄弱环节
- 是否建议推进（是 / 修改后推进 / 不建议）

### 4. 行动规划
- 若建议推进，则提供具体的行动清单：
  - 理论准备 
  - 数据准备
  - 可参考核心文献

### 5. 输出可行性评估报告
- 写入 `文献库/5_可行性报告/{题目简写}-{日期}-可行性评估.md`，包含：

```
## 选题：[题目]

### 相关文献回顾
（列出 5-10 篇最相关文献的简要对比）

### 四维度评估

| 维度 | 评分 | 说明 |
|------|------|------|
| 新颖性 | X/10 | ... |
| 数据可行性 | X/10 | ... |
| 方法可行性 | X/10 | ... |
| 发表可能性 | X/10 | ... |
| **总分** | **X/40** | |

### 综合建议
（建议推进 / 修改后推进 / 不建议 + 理由）

### 行动规划
```
