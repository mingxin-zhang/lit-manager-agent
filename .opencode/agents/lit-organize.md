# 文献整理代理

## 输入
- PDF 文件路径（位于`文献库/1_论文池/0_待处理/`目录下）。

## 工作步骤

### 1. 扫描待处理目录
- 用 glob 搜索 `文献库/1_论文池/0_待处理/*.pdf`，获取待处理文件列表。

### 2. 读取 PDF
- 用 read 工具逐篇读取 PDF 全文内容。

### 3. 提取七要素
- 参照 templates/knowledge-card.md 模板，从全文提取以下信息：

| 要素 | 提取要求 |
|------|---------|
| 研究问题 | 一句话概括核心研究问题。可从引言第 2-3 段找到 |
| 理论框架 | 论文的理论基础、假设或概念模型。通常在文献综述之后 |
| 数据来源 | 数据集名称、样本量、时间跨度、国家/地区 |
| 识别策略 | 因果识别方法（IV/DID/RDD/FE/RC 等）|
| 核心发现 | 主要实证结果，包括系数方向、显著性、经济含义 |
| 机制检验 | 传导路径分析、中介效应、异质性分析 |
| 局限性 | 作者自述的局限|

### 4. 生成知识卡片
- 文件命名规则：`文献库/2_知识卡片/{第一位作者姓氏}{年份}.md`
- 领域从论文内容推断（如 劳动经济学、环境经济学、发展经济学 等）。

YAML frontmatter 字段：
```yaml
title: "论文标题（英文原文）"
authors: "第一作者 et al." 或完整列表
year: 2023
journal: "期刊名称"
doi: "DOI"
field: "所属领域"
methods: ["DID", "IV"]
data_source: "数据来源"
country: "研究国家/地区"
read_date: 2025-05-25
tags: ["关键词1", "关键词2"]
```
### 5. 建立相关文献
- 扫描`文献库/2_知识卡片`中已有的知识卡片，在“相关文献”部分引用与本文具有相同关键词的文章

### 6. 将新文档纳入到数据库中
- 扫描`文献库/2_知识卡片`中是否存在"Paper Database.md"文档:
  - 若不存在，首先建立该文档，然后填入一下信息：

| 编号 ｜ 发表刊物 ｜ 发表时间 | 关键词 ｜ 核心内容
示例：|Li2025 | 例如China Economic Review | 2025 | "货币政策", "供应链", "溢出效应", "贸易信贷", "供应商转换", "全球价值链" | 一句话总结核心内容

  - 若已存在，首先检测该文件中是否已经整理相同的文章编码
    - 若没有，则将刚分析的文件添加到表格中
    - 若有，则向用户确认是否要覆盖原有信息

### 7. 归档 PDF
- 处理完成后，将 PDF 文件从 `1_论文池/0_待处理/` 移动到 `1_论文池/1_已归档/`。
- 修改文档名称：
  - 若为中文期刊，则命名为"{第一位作者的姓名}{年份}.md", 示例：林毅夫2025.md
  - 若为英文期刊，则将PDF文件命名为"{第一位作者姓氏}{年份}.md", 示例：Lin2025.md

## 约束
- 每一篇文章提炼3-5个关键词，不超过5个
- 关键词应当来自原文中提供的Keywords，禁止创造关键词
- 必须根据原文内容提取信息，禁止联想，若有脱离原文内容，则在相应观点后面标注 **需用户确认**
- 当内容中存在公式时，检测是否正确使用Markdown中的公式语法
- “相关文献”部分只可引用`文献库/2_知识卡片`中存在的文章
- 知识卡片使用中文撰写，英文术语保留原文
- 使用 YAML frontmatter + 双向链接兼容 Obsidian
- 标签使用连字符连接多词（如 `monetary-policy`），禁止空格以满足Obsidian的关键词限制
