# Literature Manager Skill

## 触发条件
- "搜索关于 / 帮我找一下 / 查一下 [关键词] 的文献"
- "整理 / 分析 / 处理 / 导入 / 梳理 这篇论文"
- "分析搜索结果 / 分析我的文献库 / 最近读了什么 / 有什么趋势"
- "帮我选题 / 推荐几个题目 / 有什么可以做的 / 有什么值得做的"
- "评估这个题目 / 这个选题怎么样 / 值得做吗 / 有没有价值 / 有没有问题"

## 路由规则
| 用户意图 | 调用的子代理 | 数据前置条件 | 主代理前置交互 |
|---------|-------------|------------|--------------|
| 搜索文献 | lit-search | 无 | 逐项确认：关键词（含拓展词）、年份范围、期刊层级 → 调用|
| 整理文献 | lit-organize | `1_论文池/0_待处理/`中有PDF | 确认目标 PDF → 调用 |
| 分析文献 | lit-analyze | `2_知识卡片/`中有md或`0_搜索结果/`中有CSV | 确认数据源：搜索结果/知识卡片/两者 → 调用|
| 辅助选题 | lit-topic | `3_分析报告/`中有分析报告 | 确认数据源：分析报告/搜索结果/知识卡片/全部 → 调用 |
| 评估题目 | lit-evaluate | 用户选定题目 | 确认题目措辞 → 调用 |

## 主代理原则
- 调用子代理时，prompt 只传递数据类型（如"知识卡片"或"搜索结果"）和文件路径，不过度规定分析范围。让子代理自行执行其内部的用户交互流程（如 AskUserQuestions），主代理不能替子代理跳过这些步骤。

## 对话模式示例
- 用户："帮我搜索最低工资对就业影响的文献"
  - 先向用户逐项确认：关键词（含拓展词）、年份范围、期刊层级，确认完毕后再调用 lit-search

- 用户："帮我整理一下这篇论文"（附带 PDF 或路径）
  - 确认 PDF 在 `1_论文池/0_待处理/` 中，调用 lit-organize

- 用户："最近我的研究趋势怎么样？"
  - 确认数据源：搜索结果 / 知识卡片 / 两者，确认后调 lit-analyze

- 用户："分析供应链主题文献的情况"
 - 确认数据源：搜索结果 / 知识卡片 / 两者，确认后调 lit-analyze

- 用户："基于分析结果帮我推荐 3 个题目"
  - 确认数据源：分析报告 / 搜索结果 / 知识卡片 / 全部，再调用 lit-topic

- 用户："评估一下'最低工资对中国制造业就业的影响'这个题目"
  - 确认题目措辞，调用 lit-evaluate
