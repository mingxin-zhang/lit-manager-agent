Sub-agents — Part 1 of 3

子代理详解 (1/3)

Step 1

🔍 lit-search · 文献搜索

输入关键词 + 年份范围 + 期刊层级
工具CrossRef REST API 原生关键词搜索
排序score 关联度评分，无需 LLM 二次排序
约束关键词 ≤ 5；必含原始关键词；期刊可选 top5 / field_top / all
输出文献库/0_搜索结果/{关键词}-{范围}-{日期}.csv
Step 2

📝 lit-organize · 文献整理

输入PDF（待处理目录）
核心从全文提取「七要素」
七要素研究问题 / 理论框架 / 数据来源 / 识别策略 / 核心发现 / 机制检验 / 局限性
格式YAML frontmatter + Obsidian 双向链接兼容
输出文献库/2_知识卡片/{姓氏}{年份}.md
Sub-agents — Part 2 of 3

子代理详解 (2/3)

Step 3

📊 lit-analyze · 文献分析

数据源搜索结果 CSV 或 知识卡片 MD
统计发表总量 / 主题分布 / 期刊分布 / 时间趋势
演进年度热点转移 / 上升 vs 下降主题 / 新交叉领域
深度明星学者 / 矛盾观点 / 方法论分歧
空白话题空白 / 理论空白 / 方法空白 / 情境空白
输出文献库/3_分析报告/{关键词}-{日期}-分析报告.html
Step 4

💡 lit-topic · 辅助选题

输入lit-analyze 生成的分析报告
生成矛盾调和 + 空白填补 + 交叉嫁接
筛选差异化 / 数据可获取 / 方法可行 / 有政策含义
选题结构中英文题目 + 核心问题 + 创新点 + 数据 + 识别策略 + 期刊层次
输出文献库/4_选题报告/{关键词}-{日期}-选题报告.md
Sub-agents — Part 3 of 3

子代理详解 (3/3)

Step 5

✅ lit-evaluate · 可行性评估

输入用户选定的研究题目
维度 1 新颖性 — 与已有文献差异 · 新问题/新机制/新视角
维度 2 数据可行性 — 数据存在性 · 获取难度 · 样本量 · 数据质量
维度 3 方法可行性 — 识别策略 · 内生性威胁 · 稳健性空间 · 能力匹配
维度 4 发表可能性 — 话题热度 · 边际贡献 · 政策相关性 · 时机
输出 文献库/5_可行性报告/{题目}-{日期}-可行性评估.md

Design Principles

设计 原则

六大原则贯穿 Skill 与 Sub-agent 的双层设计

🔒

强制确认规则

搜索前逐项确认关键词、年份、期刊层级；评估前确认目标期刊。主代理不替子代理跳过用户交互。

🧩

关注点分离

Skill 层负责路由和触发，Sub-agent 层负责执行。每个子代理独立封装完整规范。

📐

结构化输出

CSV（搜索结果）→ YAML+MD（知识卡片）→ HTML（分析报告）→ 四维度评分表（评估）。

🔄

可串联工作流

搜索 → 整理 → 分析 → 选题 → 评估，每步输出作为下一步输入，也可独立调用。

🛡️

约束自检

关键词 ≤ 5 且必含原始词；标签用连字符；不联想不创造；引用文献必须存在于库中。

🎯

领域聚焦

劳动经济学、发展经济学、环境经济学、供应链管理、数字经济等应用微观计量领域。
